# 语雀官方接口说明

## 1. 文档定位

本文基于以下官方资料整理：

- `https://www.yuque.com/yuque/developer/api`
- `https://www.yuque.com/yuque/developer/openapi`
- `https://www.yuque.com/yuque/developer/vzippmige58g7r9t`
- `https://www.yuque.com/raw?filekey=yuque%2F0%2F2025%2Fyaml%2F35160816%2F1763712767741-dfc6b260-2c10-41ac-8064-275eb9bf7c39.yaml`

目标是提供一份**更适合程序开发**的接口说明，便于后续直接据此编写代码。

当前整理基于本地 YAML：

- OpenAPI: `3.1.0`
- API version: `2.0.1`
- Server: `https://www.yuque.com`

补充说明：

- 当前项目所需的知识库、目录、文档读取能力，都可以通过 API 覆盖。
- 语雀附件导出能力目前没有稳定、直接的官方接口可供本项目使用，因此当前版本默认保留附件原始链接。

## 2. 接入总览

### 2.1 基础信息

- 基础域名：`https://www.yuque.com`
- 认证方式：`X-Auth-Token` 请求头
- 鉴权类型：`apiKey in header`
- 数据格式：JSON
- OpenAPI 文档支持导入 Postman 等工具

### 2.2 官方限流

根据官方页面说明：

- 每小时最多 `5000` 次请求
- 每秒最多 `100` 次请求
- 响应头会返回：
  - `X-RateLimit-Limit`
  - `X-RateLimit-Remaining`
- 个人或团队下的所有 Token **共享同一套限额**

### 2.3 常见错误码

YAML 中定义的通用响应：

| 状态码 | 含义 |
|---|---|
| 400 | 请求参数非法 |
| 401 | Token / Scope 未通过鉴权 |
| 403 | 无操作权限 |
| 404 | 实体未找到 |
| 422 | 请求参数校验失败 |
| 429 | 访问频率超限 |
| 500 | 内部错误 |

## 3. 认证与权限模型

### 3.1 请求头

```http
X-Auth-Token: <your_token>
```

### 3.2 Token 类型

根据官方 Overview 页面，可确认存在两类 Token：

#### 个人 Token

- 从个人设置页面获取
- 权限范围由当前用户本身权限决定
- 适合个人知识库、个人可访问团队知识库的读取或写入

#### 团队 Token

- 从团队设置页面获取
- 仅旗舰版空间可使用
- 可获取该团队内的知识库、文档、成员等相关数据

### 3.3 权限注意事项

官方公告说明：个人 Token 的访问范围已收紧。

实际开发时应遵循：

- 不要假设 Token 可以访问整个空间
- 读取文档前要假设可能发生 `401/403/404`
- 批量同步空间内容时，优先使用明确具备权限的 Token

## 4. URL 标识体系

官方明确给出 URL 结构：

```text
https://www.yuque.com/yuque/developer/api
+-- yuque/      [group_login]
+-- developer/  [book_slug]
+-- api         [doc_slug]
```

### 4.1 核心标识字段

| 字段 | 含义 |
|---|---|
| `group_login` | 团队或用户登录名 |
| `book_slug` | 知识库路径 |
| `doc_slug` | 文档路径 |
| `book_id` | 知识库 ID |
| `doc_id` | 文档 ID |

### 4.2 两套定位方式

语雀大量接口支持两种资源定位方式：

#### 按路径访问

适合已知 URL 结构的场景：

- `group_login`
- `book_slug`
- `doc_slug`

示例：

```http
GET /api/v2/repos/{group_login}/{book_slug}/docs/{doc_slug}
```

#### 按 ID 访问

适合已经建立本地映射、需要更稳定更高效访问的场景：

- `book_id`
- `doc_id`

示例：

```http
GET /api/v2/repos/{book_id}/docs/{doc_id}
```

### 4.3 开发建议

- 首次从 URL 入手时，优先使用 path 型接口
- 建立本地索引后，后续读写优先使用 ID 型接口
- 对外展示可保留 slug，对内存储建议同时保存 id + slug

## 7. 用户类接口

### 7.1 心跳

```http
GET /api/v2/hello
```

用途：

- 检查 Token 是否可用
- 检查 API 服务连通性

### 7.2 获取当前 Token 的用户详情

```http
GET /api/v2/user
```

用途：

- 获取当前 Token 对应身份
- 初始化登录上下文
- 作为后续拉取团队、知识库的起点

### 7.3 获取用户的团队

```http
GET /api/v2/users/{id}/groups
```

路径参数：

- `id`: 用户 login 或 ID

查询参数：

- `role`: 角色筛选，`0=管理员`，`1=成员`
- `offset`: 偏移量，默认 `0`

用途：

- 列出用户可见团队
- 建立“用户 -> 团队 -> 知识库”的同步链路

## 8. 搜索接口

### 8.1 通用搜索

```http
GET /api/v2/search
```

必填参数：

- `q`: 搜索关键词，最大长度 `200`
- `type`: 搜索类型，`doc` 或 `repo`

可选参数：

- `scope`: 搜索范围
- `page`: 页码，`1-100`
- `offset`: 旧分页字段，已废弃
- `creatorId`: 按作者 ID 搜索，已废弃
- `creator`: 按作者 login 搜索

### 8.2 scope 规则

官方给出的范围示例：

- 搜索团队中的文档：`scope=group_a` 且 `type=doc`
- 搜索团队中的知识库：`scope=group_a` 且 `type=repo`
- 搜索某个知识库中的文档：`scope=group_a/book_x` 且 `type=doc`

### 8.3 开发建议

- 全局同步前，可先用搜索做定向发现
- 用户输入搜索词时，应限制长度并做好 URL encode
- 优先使用 `page`，不要再使用废弃的 `offset`

## 9. 团队成员接口

### 9.1 获取团队成员

```http
GET /api/v2/groups/{login}/users
```

路径参数：

- `login`: 团队 login 或 ID

查询参数：

- `role`: 角色筛选，`0=管理员`，`1=成员`，`2=只读成员`
- `offset`: 偏移量，默认 `0`

### 9.2 变更成员

```http
PUT /api/v2/groups/{login}/users/{id}
```

路径参数：

- `login`: 团队 login 或 ID
- `id`: 用户 login 或 ID

请求体：

```json
{
  "role": 1
}
```

`role` 枚举：

- `0`: 管理员
- `1`: 成员
- `2`: 只读成员

### 9.3 删除成员

```http
DELETE /api/v2/groups/{login}/users/{id}
```

用途：

- 团队成员同步
- 成员角色变更
- 后台管理能力接入

## 10. 知识库 repo 接口

repo 接口既支持“用户/团队 + slug”，也支持“book_id”。

### 10.1 获取知识库列表

用户：

```http
GET /api/v2/users/{login}/repos
```

团队：

```http
GET /api/v2/groups/{login}/repos
```

查询参数：

- `offset`: 偏移量，默认 `0`
- `limit`: 每页数量，最大 `100`，默认 `100`
- `type`: 知识库类型
  - `Book`: 文档型知识库
  - `Design`: 画板型知识库
- `filterByAbility`: 权限筛选
  - `create_doc`: 仅返回具备创建文档权限的知识库

### 10.2 创建知识库

用户：

```http
POST /api/v2/users/{login}/repos
```

团队：

```http
POST /api/v2/groups/{login}/repos
```

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `name` | string | 是 | 名称 |
| `slug` | string | 是 | 路径 |
| `description` | string | 否 | 简介 |
| `public` | integer | 否 | `0=私密,1=公开,2=企业内公开` |
| `enhancedPrivacy` | boolean | 否 | 增强私密性 |

### 10.3 获取知识库详情

按 ID：

```http
GET /api/v2/repos/{book_id}
```

按路径：

```http
GET /api/v2/repos/{group_login}/{book_slug}
```

### 10.4 更新知识库

按 ID：

```http
PUT /api/v2/repos/{book_id}
```

按路径：

```http
PUT /api/v2/repos/{group_login}/{book_slug}
```

可更新字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | string | 名称 |
| `slug` | string | 路径 |
| `description` | string | 简介 |
| `public` | integer | 公开性 |
| `toc` | string | 目录 Markdown，可用于批量更新目录 |

关于 `toc`：

- 可批量更新知识库目录
- 必须是 Markdown 格式
- 示例形态：`[名称](文档路径)`

### 10.5 删除知识库

按 ID：

```http
DELETE /api/v2/repos/{book_id}
```

按路径：

```http
DELETE /api/v2/repos/{group_login}/{book_slug}
```

## 11. 文档 doc 接口

### 11.1 获取知识库的文档列表

按 ID：

```http
GET /api/v2/repos/{book_id}/docs
```

按路径：

```http
GET /api/v2/repos/{group_login}/{book_slug}/docs
```

查询参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| `offset` | integer | 偏移量，默认 `0` |
| `limit` | integer | 每页数量，最大 `100` |
| `optional_properties` | string | 附加字段，逗号分隔 |

`optional_properties` 支持：

- `hits`: 阅读数
- `tags`: 标签
- `latest_version_id`: 最新已发版本 ID

注意：

- 当每页数量超过 `100` 时，该字段会失效

### 11.2 获取文档详情

通用按文档 ID：

```http
GET /api/v2/repos/docs/{id}
```

按知识库 + 文档：

```http
GET /api/v2/repos/{book_id}/docs/{id}
GET /api/v2/repos/{group_login}/{book_slug}/docs/{id}
```

这里的 `id` 支持：

- 文档 ID
- 文档路径（slug）

额外查询参数：

- `page_size`: 数据表分页大小，`1-200`，默认 `100`
- `page`: 数据表页码，默认 `1`

这说明：

- 语雀文档详情接口已兼容“数据表型内容”分页访问
- 后续写 SDK 时，详情接口应保留 `page` / `page_size` 能力

### 11.3 创建文档

按 ID：

```http
POST /api/v2/repos/{book_id}/docs
```

按路径：

```http
POST /api/v2/repos/{group_login}/{book_slug}/docs
```

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `slug` | string | 否 | 路径 |
| `title` | string | 否 | 标题，默认 `无标题` |
| `public` | integer | 否 | `0=私密,1=公开,2=企业内公开` |
| `format` | string | 否 | `markdown/html/lake`，默认 `markdown` |
| `body` | string | 是 | 正文内容 |

补充：

- `public` 不填时，继承知识库公开性
- `format` 支持：
  - `markdown`
  - `html`
  - `lake`

### 11.4 更新文档

按 ID：

```http
PUT /api/v2/repos/{book_id}/docs/{id}
```

按路径：

```http
PUT /api/v2/repos/{group_login}/{book_slug}/docs/{id}
```

可更新字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `slug` | string | 路径 |
| `title` | string | 标题 |
| `public` | integer | 公开性 |
| `format` | string | `markdown/html/lake` |
| `body` | string | 正文 |

### 11.5 删除文档

按 ID：

```http
DELETE /api/v2/repos/{book_id}/docs/{id}
```

按路径：

```http
DELETE /api/v2/repos/{group_login}/{book_slug}/docs/{id}
```

### 11.6 文档历史版本

获取版本列表：

```http
GET /api/v2/doc_versions?doc_id={doc_id}
```

获取版本详情：

```http
GET /api/v2/doc_versions/{id}
```

用途：

- 增量同步
- 审计与回滚辅助
- 历史比对

## 12. 目录 TOC 接口

### 12.1 获取目录

按 ID：

```http
GET /api/v2/repos/{book_id}/toc
```

按路径：

```http
GET /api/v2/repos/{group_login}/{book_slug}/toc
```

### 12.2 更新目录

按 ID：

```http
PUT /api/v2/repos/{book_id}/toc
```

按路径：

```http
PUT /api/v2/repos/{group_login}/{book_slug}/toc
```

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `action` | string | 是 | `appendNode/prependNode/editNode/removeNode` |
| `action_mode` | string | 否 | `sibling/child` |
| `target_uuid` | string | 否 | 目标节点 UUID，默认根节点 |
| `node_uuid` | string | 否 | 操作节点 UUID |
| `doc_id` | integer | 否 | 文档 ID，已废弃 |
| `doc_ids` | array | 否 | 文档 ID 数组 |
| `type` | string | 否 | `DOC/LINK/TITLE`，默认 `DOC` |
| `title` | string | 否 | 节点名称 |
| `url` | string | 否 | 外链地址 |
| `open_window` | integer | 否 | `0=当前页,1=新窗口` |
| `visible` | integer | 否 | `0=不可见,1=可见` |

### 12.3 目录操作语义

`action` 取值：

- `appendNode`: 尾插
- `prependNode`: 头插
- `editNode`: 编辑节点
- `removeNode`: 删除节点

`action_mode` 取值：

- `sibling`: 同级
- `child`: 子级

节点类型：

- `DOC`: 文档
- `LINK`: 外链
- `TITLE`: 分组

### 12.4 目录接口注意事项

YAML 中明确说明：

- 创建场景下不支持同级头插 `prependNode`
- 删除节点不会删除关联文档
- `removeNode + sibling` 表示删除当前节点
- `removeNode + child` 表示删除当前节点及子节点
- `target_uuid` / `node_uuid` 需要先通过“获取目录”接口拿到

### 12.5 开发建议

目录同步是后续代码实现的重点，因为它能表达：

- 文档树结构
- 分组节点
- 外链节点
- 可见性信息

如果后续要做“语雀导出为本地目录结构”，建议先拉：

1. repo 详情
2. toc
3. docs 列表 / docs 详情

## 13. 统计 statistic 接口

统计接口都在团队维度下。

### 13.1 团队汇总统计

```http
GET /api/v2/groups/{login}/statistics
```

### 13.2 团队成员统计

```http
GET /api/v2/groups/{login}/statistics/members
```

参数：

- `name`: 成员名过滤
- `range`: `0/30/365`
- `page`: 默认 `1`
- `limit`: 最大 `20`，默认 `10`
- `sortField`: `write_doc_count/write_count/read_count/like_count`
- `sortOrder`: `desc/asc`

### 13.3 团队知识库统计

```http
GET /api/v2/groups/{login}/statistics/books
```

参数：

- `name`: 知识库名过滤
- `range`: `0/30/365`
- `page`
- `limit`
- `sortField`:
  - `content_updated_at_ms`
  - `word_count`
  - `post_count`
  - `read_count`
  - `like_count`
  - `watch_count`
  - `comment_count`
- `sortOrder`: `desc/asc`

### 13.4 团队文档统计

```http
GET /api/v2/groups/{login}/statistics/docs
```

参数：

- `bookId`: 指定知识库
- `name`: 文档名过滤
- `range`: `0/30/365`
- `page`
- `limit`
- `sortField`:
  - `content_updated_at`
  - `word_count`
  - `read_count`
  - `like_count`
  - `comment_count`
  - `created_at`
- `sortOrder`: `desc/asc`

用途：

- 运营报表
- 内容活跃度分析
- 团队成员贡献统计

## 14. 常用开发流程

### 14.1 校验 Token 是否可用

推荐顺序：

1. `GET /api/v2/hello`
2. `GET /api/v2/user`

### 14.2 拉取某个用户或团队下的知识库

1. `GET /api/v2/users/{login}/repos`
2. 或 `GET /api/v2/groups/{login}/repos`

### 14.3 拉取知识库完整内容

推荐顺序：

1. 获取知识库详情
2. 获取 TOC
3. 获取文档列表
4. 逐个获取文档详情
5. 如需增量同步，再补历史版本接口

### 14.4 创建或更新文档

1. 获取目标知识库
2. 确认权限
3. `POST /docs` 或 `PUT /docs/{id}`
4. 如需维护目录结构，再调用 TOC 更新接口

### 14.5 本项目最相关的接口

如果后续要做“语雀转 Markdown / 本地导出”，优先使用：

- `GET /api/v2/user`
- `GET /api/v2/users/{login}/repos`
- `GET /api/v2/groups/{login}/repos`
- `GET /api/v2/repos/{group_login}/{book_slug}`
- `GET /api/v2/repos/{group_login}/{book_slug}/toc`
- `GET /api/v2/repos/{group_login}/{book_slug}/docs`
- `GET /api/v2/repos/{group_login}/{book_slug}/docs/{id}`
- `GET /api/v2/doc_versions`

## 15. 请求示例

### 15.1 curl：获取当前用户

```bash
curl -H "X-Auth-Token: YOUR_TOKEN" \
  "https://www.yuque.com/api/v2/user"
```

### 15.2 curl：获取知识库文档列表

```bash
curl -H "X-Auth-Token: YOUR_TOKEN" \
  "https://www.yuque.com/api/v2/repos/yuque/developer/docs?limit=100"
```

### 15.3 curl：获取文档详情

```bash
curl -H "X-Auth-Token: YOUR_TOKEN" \
  "https://www.yuque.com/api/v2/repos/yuque/developer/docs/api"
```

### 15.4 curl：创建文档

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -d '{
    "title": "测试文档",
    "slug": "test-doc",
    "format": "markdown",
    "body": "# Hello Yuque"
  }' \
  "https://www.yuque.com/api/v2/repos/yuque/developer/docs"
```

### 16.2 resource 层

建议拆分：

- `users`
- `repos`
- `docs`
- `toc`
- `search`
- `statistics`

### 16.3 model 层

建议建模：

- `User`
- `Group`
- `Repo`
- `Doc`
- `TocNode`

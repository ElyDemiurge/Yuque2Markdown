# 语雀官方接口说明

## 1. 用途和来源

这份文档只记录 Yuque2Markdown v0.4.3 导出流程会用到的接口和限制。未接入的写接口、团队成员、统计、搜索和历史版本接口不再展开，避免文档和实现脱节。

官方资料：

- `https://www.yuque.com/yuque/developer/api`
- `https://www.yuque.com/yuque/developer/openapi`
- `https://www.yuque.com/yuque/developer/vzippmige58g7r9t`
- `https://www.yuque.com/raw?filekey=yuque%2F0%2F2025%2Fyaml%2F35160816%2F1763712767741-dfc6b260-2c10-41ac-8064-275eb9bf7c39.yaml`

本地 YAML 元信息：

- OpenAPI: `3.1.0`
- API version: `2.0.1`
- Server: `https://www.yuque.com`

## 2. 当前代码实际接入的接口

### Token / OpenAPI 模式

- `GET /api/v2/user`
- `GET /api/v2/users/{login}/repos`
- `GET /api/v2/repos/{group_login}/{book_slug}`
- `GET /api/v2/repos/{group_login}/{book_slug}/toc`
- `GET /api/v2/repos/{group_login}/{book_slug}/docs`
- `GET /api/v2/repos/{group_login}/{book_slug}/docs/{id_or_slug}`

### Cookie / 网页端模式

- `GET /api/mine`
- `GET /api/mine/books`
- `GET /api/catalog_nodes?book_id=...`
- `GET /api/docs?book_id=...`
- `GET /api/docs/{id_or_slug}?book_id=...`

### 资源下载

- 图片和附件直接请求资源 URL，不走文档接口。
- 语雀附件 URL 会补 `download=1`。
- 如果下载响应像 HTML 页面，代码会按下载失败处理。

## 3. 认证与权限

Token 模式使用请求头：

```http
X-Auth-Token: <your_token>
```

Cookie 模式发送浏览器 Cookie，并改走网页端接口。个人语雀知识库的网页端鉴权主要依赖 `_yuque_session`。

注意：

- 项目只支持个人语雀知识库导出。
- 非当前登录账号的知识库会在列表中灰显，暂不支持导出。
- 企业版自定义 Cookie key 不支持。
- 语雀官方 OpenAPI 没有附件导出接口；Token 登录时附件保留原始链接，Cookie 登录时才能按扩展名下载附件。

## 4. URL 与标识

语雀知识库 URL 通常为：

```text
https://www.yuque.com/{group_login}/{book_slug}
```

项目中主要使用这些标识：

| 字段 | 用途 |
|---|---|
| `group_login` | 用户或团队 login，来自 URL 第一段 |
| `book_slug` | 知识库 slug，来自 URL 第二段 |
| `book_id` | Cookie 网页端接口使用 |
| `doc_id` | 文档数字 ID |
| `slug` | 文档短链接标识，用于内部链接改写 |

用户手动输入完整 URL 时，程序取前两段作为知识库标识；配置文件中只保存 `group_login/book_slug`。

## 5. 限流与错误

官方页面给出的限流信息：

- 每小时最多 `5000` 次请求
- 每秒最多 `100` 次请求
- 个人或团队下的所有 Token 共享限额

客户端会读取响应头中的 `X-RateLimit-Limit`、`X-RateLimit-Remaining`，并在控制台状态栏和导出进度页展示。

常见状态码：

| 状态码 | 含义 |
|---|---|
| 400 | 请求参数非法 |
| 401 | Token / Scope 未通过鉴权 |
| 403 | 无操作权限 |
| 404 | 实体未找到 |
| 422 | 请求参数校验失败 |
| 429 | 访问频率超限 |
| 500 | 内部错误 |

遇到 `429` 时，客户端会按配置等待后重试。

## 6. 项目相关接口

### `GET /api/v2/user`

用途：校验 Token，并取得当前用户 login。

项目使用字段：

- `data.login`
- `data.name`

### `GET /api/v2/users/{login}/repos`

用途：列出当前用户可访问的知识库。

项目使用字段：

- `id`
- `name`
- `slug`
- `namespace`
- `user.login`

### `GET /api/v2/repos/{group_login}/{book_slug}`

用途：取得知识库详情，补齐 `id`、`name`、`namespace` 等字段。

项目使用字段：

- `id`
- `name`
- `slug`
- `namespace`
- `user.login`

### `GET /api/v2/repos/{group_login}/{book_slug}/toc`

用途：读取目录树，生成本地目录结构和文档选择树。

项目使用字段：

- `id`
- `uuid`
- `title`
- `slug`
- `url`
- `parent_uuid`
- `type`

### `GET /api/v2/repos/{group_login}/{book_slug}/docs`

用途：读取知识库内文档列表，辅助构建 slug、id 和标题映射。

项目使用字段：

- `id`
- `slug`
- `title`
- `book_id`
- `user_id`
- `format`
- `updated_at`

### `GET /api/v2/repos/{group_login}/{book_slug}/docs/{id_or_slug}`

用途：读取单篇文档详情。

项目使用字段：

- `id`
- `slug`
- `title`
- `body_lake`
- `body`
- `content`
- `updated_at`

优先使用 `body_lake` 转 Markdown；`body_lake` 为空时依次回退到 `body`、`content`。

## 7. Cookie 网页端接口

Cookie 模式用于两件事：

- 读取当前浏览器登录账号能看到的个人知识库。
- 在导出时下载附件。

网页端接口不是 OpenAPI 的稳定承诺，语雀改页面接口后这里可能需要同步适配。相关代码集中在 `core_modules/export/client.py`。

## 8. 与当前项目实现的差异

- 当前项目没有接入创建、更新、删除接口。
- 当前项目没有接入团队成员管理、统计、搜索和版本同步接口。
- 当前项目不通过 OpenAPI 下载附件。
- Cookie 模式依赖网页端接口，不属于 OpenAPI 正式能力范围。

# 配置文件字段说明

配置文件路径：`yuque2markdown.config.json`，位于程序运行目录。

该文件由交互式控制台自动生成和管理，**不要直接编辑**，所有字段均可通过控制台菜单修改。

---

## 顶层字段

### `version`
**类型**：`int`
**默认值**：`1`
**说明**：配置文件版本号，用于标识配置格式。当前为 `1`，后续如有格式变更会递增。

---

### `token`
**类型**：`string`
**默认值**：`""`
**说明**：语雀 Personal Access Token（PAT），用于调用语雀 API。
- 程序启动后首次设置时输入，后续保存在此字段。
- 如果 `persist_token` 为 `false`，此字段会被清空。

---

### `persist_token`
**类型**：`bool`
**默认值**：`true`
**说明**：是否将 Token 持久化保存到配置文件。
- `true`：Token 写入配置文件，程序重启后自动恢复登录状态。
- `false`：Token 仅存在于当前会话，程序退出后丢失，适合公共或多用户环境下避免 Token 泄露。

---

### `last_repo_input`
**类型**：`string`
**默认值**：`""`
**说明**：上一次成功验证的知识库标识，格式为 `group_login/book_slug`。
- 用于下次启动时快速恢复上次工作状态。
- 通过控制台“知识库”菜单项设置后自动更新。

---

## `export_defaults`（导出默认配置）

### `output_dir`
**类型**：`string`
**默认值**：`"output"`
**说明**：导出文档的根目录。所有知识库的导出文件都会放在此目录下。
- 可在控制台“导出配置 -> 输出目录”中修改。
- 建议使用相对路径（相对于程序运行目录）或绝对路径。

---

### `resume`
**类型**：`bool`
**默认值**：`true`
**说明**：是否启用断点续导。
- `true`：导出时跳过已完成的文档，保留上次导出进度（基于 checkpoint 文件）。
- `false`：忽略 checkpoint，每次导出从头开始。
- 对知识库重新导出、文档内容有更新时，设为 `false`。

---

### `strict`
**类型**：`bool`
**默认值**：`false`
**说明**：严格模式。
- `true`：任一文档导出失败，立即停止整个导出流程。
- `false`：失败不影响其他文档，程序继续处理后续文档。
- 调试导出问题时可临时开启。

---

### `request_interval`
**类型**：`float`
**默认值**：`0.2`
**说明**：两次 API 请求之间的间隔秒数。语雀 API 有调用频率限制，建议保持 `0.2` 或更大。
- 值越小越快，但触发限流（429）的概率越高。
- 语雀限流后程序会自动等待后重试，无需手动降低此值。

---

### `timeout`
**类型**：`int`
**默认值**：`10`
**说明**：单次 API 请求的超时秒数（单位：秒）。
- 网络较慢时可适当增大，但一般不需要修改。
- 对应 `YuqueClient` 构造的 `timeout` 参数。

---

### `token_check_timeout`
**类型**：`int`
**默认值**：`5`
**说明**：刷新连接状态时，验证 Token 有效性的超时秒数。
- 比普通请求超时短，目的是快速判断 Token 是否已失效。

---

### `request_max_retries`
**类型**：`int`
**默认值**：`5`
**说明**：单次请求失败（网络错误）后的最大重试次数。
- 仅对网络层面的错误（如连接超时）重试。
- 语雀限流（429）不在此重试范围内，会进入等待后重试逻辑。

---

### `rate_limit_backoff_seconds`
**类型**：`float`
**默认值**：`5.0`
**说明**：触发语雀限流后，每次重试前的基础等待秒数。
- 实际等待时间 = `rate_limit_backoff_seconds × 当前重试次数`，直到达到 `max_backoff_seconds`。

---

### `network_backoff_seconds`
**类型**：`float`
**默认值**：`2.0`
**说明**：网络错误重试前的基础等待秒数。
- 实际等待时间 = `network_backoff_seconds × 当前重试次数`，直到达到 `max_backoff_seconds`。

---

### `max_backoff_seconds`
**类型**：`float`
**默认值**：`60.0`
**说明**：重试等待时间的上限秒数，避免无限等待。

---

### `max_docs`
**类型**：`int | null`
**默认值**：`null`
**说明**：每次导出最多处理的文档数。
- `null`：不限制，导出知识库内所有文档。
- 正整数 N：最多成功导出 N 篇后停止，不管知识库还有多少文档。
- 适合先导出几篇验证效果，确认无误后再全量导出。

---

### `offline_assets`
**类型**：`bool`
**默认值**：`true`
**说明**：是否将图片等在线资源下载到本地。
- `true`：文档中的图片、附件等资源会下载到 `<doc>/assets/` 目录，Markdown 中的 URL 替换为本地相对路径。
- `false`：保留原文中的在线 URL，不下载资源。

---

### `assets_dir_name`
**类型**：`string`
**默认值**：`"assets"`
**说明**：每篇文档目录下存放离线资源的子目录名称。
- 实际路径为 `<文档目录>/<assets_dir_name>/`。
- 如设为 `"files"`，资源会放在 `<文档目录>/files/` 下。

---

### `fail_on_asset_error`
**类型**：`bool`
**默认值**：`false`
**说明**：资源下载失败时是否将整篇文档标记为失败。
- `true`：任一图片/附件下载失败，该文档的导出结果视为失败，记录在 checkpoint 中。
- `false`：下载失败不影响导出，仅在 warnings 中记录失败的 URL。
- 对于必须确保资源完整性的导出场景可开启。

---

## `proxy`（代理配置）

### `enabled`
**类型**：`bool`
**默认值**：`false`
**说明**：代理开关。
- `true`：启用代理，所有请求通过 `host:port` 发送。
- `false`：禁用代理，即使设置了代理地址也不会使用。
- 可在控制台“高级设置 -> 代理”中切换。

---

### `host`
**类型**：`string`
**默认值**：`""`
**说明**：本地代理服务器地址（主机名或 IP）。
- 设置后，所有 API 请求和资源下载都会通过代理（需同时开启代理开关）。
- 常见代理软件默认地址：`127.0.0.1` 或 `localhost`。
- 留空表示不使用代理。
- 可在控制台“高级设置 -> 代理地址”中修改（代理开启后显示）。

---

### `port`
**类型**：`int`
**默认值**：`7890`
**说明**：本地代理服务器端口。
- 常见代理软件默认端口：
  - Clash: `7890`
  - V2Ray: `10808`
  - Shadowsocks: `1080`
- 可在控制台“高级设置 -> 代理端口”中修改（代理开启后显示）。

---

### `test_url`
**类型**：`string`
**默认值**：`"https://www.baidu.com"`
**说明**：测试代理连通性时使用的测试地址。
- 测试代理时不使用语雀 API，而是访问此地址验证代理是否正常工作。
- 默认使用百度，因其在国内网络环境下通常稳定可达。
- 可在控制台“高级设置 -> 代理测试地址”中修改（代理开启后显示）。
- 建议使用无需认证的公开 HTTP/HTTPS 地址。

---

## `ui_preferences`（界面偏好）

### `confirm_before_export`
**类型**：`bool`
**默认值**：`true`
**说明**：导出前是否显示确认对话框。
- `true`：导出前弹出确认界面，显示导出范围、配置、文档数量等摘要，需手动确认。
- `false`：直接开始导出，无确认步骤。

---

### `auto_save_after_export`
**类型**：`bool`
**默认值**：`true`
**说明**：导出结束后是否自动保存配置文件。
- `true`：导出完成后自动将当前配置（含 Token、知识库、导出范围等）写入配置文件。
- `false`：导出完成后保持会话状态，不写入磁盘，下次启动恢复上次保存的配置。
